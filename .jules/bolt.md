## 2025-05-23 - [Pattern] Callback Stabilization with useRef
**Learning:** In a core hook like `useOnePercent`, functions that depend on the `challenges` state (like `completeChallenge`) were being recreated on every state change. This caused all components using these functions (like `ChallengeCard`) to re-render even if they were wrapped in `React.memo`, because their function props were changing. Using a `useRef` to track the current `challenges` allows these callbacks to have a stable reference while still accessing the latest state.
**Action:** Use `useRef` for frequently changing state that is needed in callbacks, ensuring the ref is updated in a `useEffect`. Combine this with functional state updates (e.g., `setChallenges(prev => ...)`) to maintain stability and correctness.
