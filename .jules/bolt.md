## 2026-07-17 - [Optimizing React Hooks & Component Rendering Paths]
**Learning:** React custom hooks that return state updates and event handlers (like `useOnePercent`) often recreate functions due to state dependency updates, causing unnecessary rendering down the tree. Utilizing `useRef` to track the state and passing a state update function to state setters (e.g., `setChallenges(prev => ...)`) decouples action triggers from direct state values, stabilizing references completely. Furthermore, algorithmic optimizations on static arrays can turn O(K*N) operations into O(N) single pass lookups.
**Action:** Hoist static constants and helper functions outside components and hooks. Use `useRef` to maintain stable references inside custom hooks to prevent reference re-creation, and memoize key pure visual components with `React.memo` and standard display names.
